1. ¿Qué es un Dato?

Un dato es una representación simbólica (numérica, alfabética, alfanumérica, etc.) de un atributo o una característica de un objeto, evento o fenómeno. En el contexto de la informática y la tecnología, los datos son la materia prima que se procesa y se manipula para obtener información significativa.

2. ¿Qué es información?

La información es el resultado del procesamiento y la interpretación de los datos. Mientras que los datos son representaciones simbólicas de atributos o características, la información tiene significado y contexto. La información se crea cuando los datos se organizan, se analizan y se presentan de manera que sean comprensibles y útiles para los seres humanos u otros sistemas.

3. ¿Diferencia entre dato e información?

La diferencia principal entre datos e información radica en su naturaleza y utilidad:

Naturaleza:

Datos: Son representaciones simbólicas de atributos o características, como números, palabras, imágenes, etc. Información: Es el resultado del procesamiento y la interpretación de los datos, que tiene significado y contexto para los usuarios.

Utilidad:

Datos: Son la materia prima que se procesa para obtener información. Por sí solos, los datos no tienen un significado claro y pueden no ser útiles hasta que se procesan y se interpretan. Información: Es el resultado del procesamiento y la interpretación de los datos. Es útil para la toma de decisiones, la comprensión de situaciones, la comunicación de ideas y mucho más.

4. ¿Tipos de datos?

Los datos pueden clasificarse en varios tipos según su naturaleza y características. Algunas de las clasificaciones comunes son:

Datos numéricos: Son aquellos que representan valores numéricos, como números enteros, decimales o fracciones. Por ejemplo, la edad de una persona, el precio de un producto, etc.

Datos textuales: Consisten en palabras, frases o texto que representan información. Pueden incluir nombres, descripciones, comentarios, etc.

Datos categóricos: Son datos que representan categorías o clases discretas. Por ejemplo, el género de una persona (masculino, femenino), el estado civil (soltero, casado, divorciado), etc.

Datos temporales: Representan información relacionada con el tiempo o la fecha. Pueden ser fechas específicas, horas del día, períodos de tiempo, etc.

Datos geoespaciales: Son datos que tienen asociadas coordenadas geográficas. Se utilizan para representar ubicaciones en la superficie terrestre o en otros espacios geográficos.

Datos binarios: Son datos que se representan utilizando solo dos valores posibles, como 0 y 1. Se utilizan en informática para representar información de manera digital.

Datos estructurados y no estructurados: Los datos estructurados están organizados en un formato predefinido, como tablas de bases de datos, mientras que los datos no estructurados no siguen un formato específico y pueden incluir texto libre, imágenes, audio, video, etc.

5. ¿Qué es un modelado de Datos?

El modelado de datos es el proceso de crear una representación estructurada y visual de los datos y sus relaciones dentro de un sistema de información. Se utiliza para definir cómo se organizarán, almacenarán y relacionarán los datos en una base de datos o en cualquier otro sistema de almacenamiento de datos.

El modelado de datos implica la creación de modelos conceptuales, lógicos y físicos que describen los diferentes aspectos de los datos, como las entidades, los atributos, las relaciones y las restricciones. Estos modelos proporcionan un marco claro para entender la estructura de los datos y cómo se relacionan entre sí.

6. ¿Tipos de bases de Datos, según: - Variabilidad - Contenido - Modelo de base de datos

Variabilidad:

Bases de datos estáticas: Son bases de datos donde los datos no cambian con el tiempo o cambian muy raramente. La información contenida en estas bases de datos suele ser estática y predefinida.

Bases de datos dinámicas: Son bases de datos donde los datos cambian con frecuencia y se actualizan de manera regular. La información contenida en estas bases de datos puede variar en función de eventos o transacciones en tiempo real.

Contenido:

Bases de datos textuales: Contienen principalmente datos de tipo texto, como documentos, correos electrónicos, blogs, etc.

Bases de datos multimedia: Almacenan datos de diferentes tipos de medios, como imágenes, audio y video.

Bases de datos geoespaciales: Almacenan datos relacionados con ubicaciones geográficas y espaciales, como mapas, coordenadas GPS, etc.

Bases de datos genómicas: Almacenan datos relacionados con el ADN y las secuencias genéticas, utilizadas en biología y genética.

Modelo de base de datos:

Bases de datos relacionales: Utilizan el modelo relacional para organizar los datos en tablas con filas y columnas, y establecen relaciones entre las tablas mediante claves primarias y foráneas.

Bases de datos NoSQL: Utilizan modelos de datos no relacionales y están diseñadas para manejar grandes volúmenes de datos no estructurados o semiestructurados de manera eficiente. Incluyen tipos como bases de datos de documentos, bases de datos de clave-valor, bases de datos de grafos, etc.

Bases de datos orientadas a objetos: Almacenan datos en forma de objetos, lo que permite representar de manera más fiel las estructuras de datos complejas utilizadas en la programación orientada a objetos.

Bases de datos jerárquicas: Organizan los datos en una estructura de árbol, donde cada registro tiene un único padre y cero o más hijos, lo que es útil para representar datos con relaciones jerárquicas.

Bases de datos de redes: Permiten modelar datos con relaciones complejas mediante la representación de entidades y sus interconexiones en una red de nodos y enlaces.

7. ¿Qué es el modelo relacional?

El modelo relacional es un modelo de datos que organiza la información en tablas bidimensionales llamadas "relaciones". Este modelo fue propuesto por Edgar F. Codd en 1970 y se convirtió en la base para la mayoría de los sistemas de gestión de bases de datos relacionales (RDBMS).

En un modelo relacional, los datos se representan como conjuntos de filas y columnas. Cada tabla consta de filas, también conocidas como "tuplas" o "registros", y columnas, también conocidas como "atributos" o "campos". Cada fila representa una entidad única dentro de la tabla y cada columna representa un atributo específico de esa entidad.

Las relaciones entre las tablas se establecen mediante el uso de claves. Una clave primaria es un atributo o una combinación de atributos que identifica de manera única cada fila en una tabla. Una clave foránea es un atributo en una tabla que hace referencia a la clave primaria de otra tabla, estableciendo así una relación entre las dos tablas.

8. ¿Describa los elementos del Diagrama entidad Relacional?

Entidades:

Representan objetos del mundo real o conceptos del dominio de la aplicación que deben ser almacenados en la base de datos. Cada entidad se representa mediante un rectángulo en el DER. Por ejemplo, en una base de datos de una biblioteca, las entidades pueden ser "Libro", "Autor", "Usuario", etc.

Atributos:

Son las características o propiedades de una entidad. Cada atributo se representa dentro del rectángulo de la entidad, generalmente en una lista. Por ejemplo, para la entidad "Libro", los atributos pueden ser "Título", "ISBN", "Año de publicación", etc.

Claves primarias:

Son atributos (o conjuntos de atributos) que identifican de manera única cada instancia de una entidad. Se denotan generalmente subrayando el atributo o atributos que componen la clave primaria. Por ejemplo, en una tabla de "Libro", el ISBN puede ser una clave primaria.

Claves foráneas:

Son atributos en una entidad que establecen una relación con la clave primaria de otra entidad. Se representan mediante una línea que conecta el atributo de la clave foránea con la clave primaria correspondiente en otra entidad.

Relaciones:

Representan asociaciones entre entidades. Pueden ser de uno a uno, uno a muchos o muchos a muchos. Se representan mediante líneas conectando las entidades involucradas en la relación. Por ejemplo, una relación entre las entidades "Libro" y "Autor" podría representar que un autor puede escribir muchos libros.

Cardinalidad:

Indica el número de instancias de una entidad que pueden estar relacionadas con un solo registro de la entidad relacionada. Se suele indicar mediante notaciones como "1" (uno), "M" (muchos), "0..1" (cero o uno), "0..M" (cero o muchos), etc.

9. ¿Qué es una entidad?

En el contexto de los modelos de datos, una entidad representa un objeto o concepto del mundo real que se puede distinguir y describir mediante un conjunto de características o atributos. Las entidades son elementos fundamentales en el diseño de bases de datos relacionales y se utilizan para organizar y estructurar la información de manera significativa.

Una entidad puede ser cualquier cosa que tenga existencia independiente y que sea relevante para el sistema que se está modelando. Por ejemplo, en una base de datos de una biblioteca, las entidades pueden incluir libros, autores, usuarios, préstamos, etc. Cada una de estas entidades se puede identificar claramente y se puede describir mediante un conjunto de atributos que capturan sus características distintivas.

En el modelo entidad-relación (ER), las entidades se representan como rectángulos y se pueden nombrar de manera descriptiva para reflejar su significado dentro del contexto del sistema. Además, las entidades pueden tener relaciones entre sí, lo que significa que están conectadas de alguna manera en la base de datos.

10. ¿Qué es un atributo y sus tipos?

Un atributo es una característica o propiedad que describe una entidad en un modelo de datos. En esencia, los atributos son las cualidades o aspectos que ayudan a definir y distinguir una entidad de otras en una base de datos. Cada entidad puede tener uno o más atributos que representan diferentes aspectos de la entidad en cuestión.

Los atributos pueden clasificarse en varios tipos, según su naturaleza y cómo se utilizan en el contexto del modelo de datos. Aquí hay algunos tipos comunes de atributos:

Atributos simples:

Son atributos que representan una única característica de una entidad. Por ejemplo, en una entidad "Persona", un atributo simple puede ser "Nombre", "Edad", "Dirección", etc.

Atributos compuestos:

Son atributos que están formados por la combinación de múltiples sub-atributos simples. Por ejemplo, en una entidad "Dirección", un atributo compuesto puede ser "Calle", "Ciudad", "Código Postal", etc.

Atributos multivalor:

Son atributos que pueden tener múltiples valores para una misma entidad. Por ejemplo, en una entidad "Habilidad" de un empleado, puede haber un atributo multivalor "Idiomas" que represente los diferentes idiomas que habla el empleado.

Atributos derivados:

Son atributos cuyos valores se derivan o se calculan a partir de otros atributos en la base de datos. Por ejemplo, en una entidad "Empleado", un atributo derivado puede ser "Edad", que se calcula a partir de la fecha de nacimiento del empleado.

Atributos clave:

Son atributos que se utilizan para identificar de manera única cada instancia de una entidad. Los atributos clave pueden ser simples o compuestos y se utilizan como claves primarias o candidatas en la base de datos.

Atributos nulos:

Son atributos que pueden tener valores nulos o ausentes en una base de datos. Esto indica que la información para ese atributo no está disponible o no es aplicable para una determinada entidad.

11. ¿Qué es una relación?

En el contexto de las bases de datos, una relación se refiere a la asociación o conexión entre dos o más entidades. Las relaciones modelan cómo las entidades están conectadas entre sí y cómo interactúan dentro del sistema de base de datos.

Una relación se define por las interacciones que ocurren entre las entidades participantes. Estas interacciones pueden tomar diferentes formas y se pueden clasificar en varios tipos:

Relación uno a uno (1:1):

Cada instancia de una entidad está asociada con exactamente una instancia de otra entidad y viceversa. Por ejemplo, en una base de datos de empleados, cada empleado puede tener solo un supervisor directo. Relación uno a muchos (1 ):

Cada instancia de una entidad está asociada con una o más instancias de otra entidad, pero cada instancia de la segunda entidad está asociada con exactamente una instancia de la primera entidad. Por ejemplo, en una base de datos de clientes y pedidos, un cliente puede realizar varios pedidos, pero cada pedido está asociado a un solo cliente.

Relación muchos a uno (N:1):

Es lo opuesto a la relación uno a muchos. Cada instancia de una entidad está asociada con exactamente una instancia de otra entidad, pero cada instancia de la segunda entidad puede estar asociada con una o más instancias de la primera entidad. Este tipo de relación es menos común, pero a veces es útil para modelar ciertos casos.

Relación muchos a muchos (N ):

Cada instancia de una entidad puede estar asociada con una o más instancias de otra entidad, y viceversa. Este tipo de relación es común en situaciones donde múltiples instancias de una entidad pueden estar relacionadas con múltiples instancias de otra entidad. Por ejemplo, en una base de datos de estudiantes y cursos, un estudiante puede estar inscrito en varios cursos, y un curso puede tener varios estudiantes inscritos.

12. ¿Qué son las llaves primarias, foránea y sus características?

Las claves primarias y foráneas son elementos fundamentales en el diseño de bases de datos relacionales que ayudan a establecer y mantener la integridad de los datos. Aquí está una descripción de cada una junto con sus características principales:

Clave Primaria:

Definición: Una clave primaria es un conjunto de uno o más atributos que identifica de forma única cada fila o instancia en una tabla de base de datos. En otras palabras, garantiza que no haya duplicados y que cada fila sea única dentro de la tabla.

Características:

Única: Cada valor de la clave primaria debe ser único en la tabla. No nula: Los valores de la clave primaria no pueden ser nulos. Inmutable: Una vez establecida, la clave primaria no cambia para una fila dada. Mínima: La clave primaria debe ser lo más corta posible sin sacrificar la unicidad. Invariable: La clave primaria no cambia con el tiempo.

Clave Foránea:

Definición: Una clave foránea es un atributo o conjunto de atributos en una tabla que establece una relación entre esa tabla y otra tabla en la base de datos. La clave foránea generalmente hace referencia a la clave primaria de la otra tabla, lo que indica una relación entre las dos tablas.

Características:

Relacionada: La clave foránea debe hacer referencia a una clave primaria existente en otra tabla. Restringida: Los valores de la clave foránea deben ser coincidentes con los valores de la clave primaria a la que hace referencia.

Opcional: En algunas relaciones, una clave foránea puede ser nula, lo que indica que la relación es opcional.

Cascada: Pueden establecerse reglas de cascada para actualizar o eliminar automáticamente filas relacionadas en la tabla secundaria cuando se actualiza o elimina una fila en la tabla principal.

13. ¿Qué son las reglas del Negocio?

Las reglas de negocio son declaraciones o directrices que definen o regulan las operaciones, actividades y comportamientos dentro de una organización. Estas reglas son diseñadas para guiar y controlar las decisiones y acciones realizadas por las personas que operan en el contexto de la organización, ya sea en el ámbito de los procesos empresariales, las operaciones diarias o la interacción con los clientes y socios.

Las reglas de negocio pueden abarcar una amplia variedad de áreas dentro de una organización, como:

Operaciones comerciales: Establecen cómo se llevan a cabo las actividades comerciales y los procesos operativos de la organización. Por ejemplo, reglas sobre cómo se gestionan los pedidos, cómo se facturan los productos o servicios, etc.

Políticas de la empresa: Definen las políticas, normas o principios que guían el comportamiento y las decisiones de los empleados. Por ejemplo, políticas de seguridad de la información, políticas de recursos humanos, etc.

Requisitos regulatorios: Establecen los requisitos legales, normativos o regulatorios que la organización debe cumplir en su funcionamiento. Por ejemplo, normativas de protección de datos, requisitos de cumplimiento financiero, etc.

Reglas de validación de datos: Especifican los criterios que los datos deben cumplir para ser considerados válidos y precisos. Por ejemplo, reglas que validan la integridad de los datos introducidos en un sistema.

Reglas de calidad: Definen los estándares de calidad que deben cumplir los productos o servicios de la organización. Por ejemplo, reglas sobre el control de calidad en la producción, la atención al cliente, etc.

14. ¿Qué es la cardinalidad o mapeo?

La cardinalidad o mapeo se refiere a la relación entre las tablas de una base de datos relacional. Especifica cuántas tuplas (filas) de una tabla están asociadas con cuántas tuplas de otra tabla.

La cardinalidad en bases de datos se clasifica generalmente en tres tipos principales:

Uno a uno (1:1): Cada fila de una tabla se asocia con exactamente una fila de otra tabla y viceversa. Este tipo de relación es menos común, pero puede ser útil en situaciones donde los datos están muy normalizados o para dividir datos sensibles en tablas separadas.

Uno a muchos (1 ): Cada fila de una tabla se puede asociar con múltiples filas de otra tabla, pero cada fila de la segunda tabla solo se puede asociar con una fila de la primera tabla. Por ejemplo, en una relación de cliente a pedido, un cliente puede realizar múltiples pedidos, pero cada pedido está asociado con un solo cliente.

Muchos a muchos (N ): Cada fila de una tabla puede estar asociada con múltiples filas de otra tabla, y viceversa. Este tipo de relación generalmente requiere una tabla de unión (tabla de intersección o tabla de relación) para vincular las filas de ambas tablas. Por ejemplo, en una relación entre estudiantes y cursos, un estudiante puede inscribirse en múltiples cursos y cada curso puede tener múltiples estudiantes.
